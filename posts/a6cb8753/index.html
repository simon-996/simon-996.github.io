为什么不用记事本？

从技术上讲，HTML 只是纯文本文件，用记事本开发确实可以运行。 但是，记事本缺乏智能提示、代码高亮和自动格式化功能，开发效率极低且容易出错。工欲善其事，必先利其器，强烈建议大家后续开发使用专业的代码编辑器。

Visual Studio Code(VS Code)

定位：全球最流行的轻量级通用代码编辑器。

特点：可以理解为“升级版的记事本 + 超级代码助手”，支持几乎所有编程语言。优点

生态无敌：后续学习网页开发进阶技术或其他语言，都有很好的插件支持

社区活跃：90% 的问题网上都有教程和解决方案

免费开源：完全免费且由微软维护，更新频率高

缺点

配置门槛高：刚下载时和记事本差不多，需要自己安装配置插件

HbuilderX

国产IDE

优点

  • 开箱即用：安装完成后无需配置插件即可使用

  • 极致中文：所有菜单、提示和报错信息都是本土化中文

  • 特色功能：后续如果要学习 uniapp，几乎只能用这个IDE

缺点

生态封闭：插件体系不如 VS Code 开放，高级功能绑定 DCloud

国际通用性差：国外或外企，几乎没人使用

广告/推广：免费版本偶尔有推广弹窗

安装以下插件让 vscode 更好用：

  1. Live Server

    • 作用：启动一个本地服务器。

    • 核心功能：当你保存代码（Ctrl + S）时，浏览器自动刷新，无需手动按 F5！

    • 重要性：⭐⭐⭐⭐⭐

  2. Open in Browser

    • 作用：允许你在代码文件上点击右键，选择 Open with Default Browser 直接在浏览器打开。

      • 重要性：⭐⭐⭐⭐

  3. Auto Rename Tag

    • 作用：智能成对修改标签。
    • 场景：当你修改开始标签 <div> 为 <section> 时，结束标签会自动同步修改，防止漏改导致页面崩溃。
    • 重要性：⭐⭐⭐⭐⭐

  4. Prettier - Code formatter

    • 作用：代码格式化工具。
    • 核心功能：每次保存（Ctrl + S）时，自动将混乱的代码缩进、空格整理得整整齐齐。
    • 重要性：⭐⭐⭐⭐⭐
