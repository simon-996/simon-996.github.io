html-css-3-表单标签

发表于2026-03-30|更新于2026-04-03|课件
|浏览量:

🎯 学习目标

  1. 理解表单的作用与应用场景。
  2. 掌握表单核心容器 <form> 常用属性。
  3. 熟练使用常用表单控件（输入框、单选、复选、下拉、文本域、按钮）。
  4. 掌握表单label绑定、表单分组，提升用户体验与代码语义化。

表单简介

什么是表单？

表单是网页收集用户信息、实现前后端交互的核心模块。

常见场景：登录注册、搜索框、留言评论、问卷调查、订单提交等。

核心结构

整个表单被 <form> 标签包裹，内部放置各类表单控件（输入框、单选框、按钮等）。

表单容器 form 标签

基本语法

属性 作用 说明
action 数据提交地址 后端接口 / 处理页面地址，表单数据发送到该地址
method 提交方式 常用：get / post
name 表单名称 用于 JS / 后端识别表单

💡 小贴士：

  • get：参数拼接在 URL 里，不安全、长度有限、适合查询
  • post：请求体传递数据，更安全、无长度限制，适合登录 / 注册 / 提交信息

表单通用属性

大部分表单输入控件共用核心属性：

属性 作用
name 必须有，后端接收数据的参数名
value 控件默认值 / 提交的值
placeholder 输入框提示文字（灰色占位符）
checked 单选 / 复选默认选中
selected 下拉选项默认选中
disabled 禁用控件，无法输入 / 选择

常用表单 input 控件

<input> 是单标签，通过 type 属性区分不同控件类型。

单行文本输入框 text

常见属性：

  • autocomplete ：用于控制浏览器表单的自动填充行为，决定是否根据用户历史输入自动填充。取值on/off
  • accesskey：使元素获得焦点的快捷键

密码框 password

输入内容隐藏，适合密码输入

单选按钮 radio

⚠️ 同一组单选必须 name 相同，实现互斥选择

复选框 checkbox

可多选，常用于爱好、协议勾选

提交按钮 submit

点击触发 form 表单提交动作

重置按钮 reset

一键清空表单所有输入内容

普通按钮 button

配合 JS 实现自定义点击功能，默认无表单行为

文件上传 file

上传图片、文档等文件

范围滑块 range

用于选择一个数值范围，如音量、亮度调节。

min：最小值

max：最大值

value：初始值

颜色拾取器 color

用于选择颜色，点击后弹出系统颜色面板。

  • value：默认颜色值（十六进制）

下拉菜单 select + option

基本结构

双标签组合，实现下拉选择列表

  • <select>：下拉容器
  • <option>：下拉选项
  • selected：设置默认选中项

多行文本域 textarea

大段文字输入（留言、简介、备注），双标签

  • rows：显示行数
  • cols：显示宽度

语义优化 label 标签

作用

点击文字即可选中输入框 / 单选 / 复选，提升用户体验 + 语义化

写法 1：for + id 绑定（推荐）

写法 2：嵌套绑定

表单分组 fieldset + legend

对多个表单模块进行分组，结构更清晰、语义化更强

表单开发易错点总结

  1. 所有需要提交数据的控件必须加 name 属性，否则后端接收不到；
  2. 同组单选 radio name 值必须一致才能互斥；
  3. label 绑定提升体验；
  4. <form> 不写 action 默认提交到当前页面；
  5. 密码框只是视觉隐藏，传输不安全，后端需加密处理；
  6. 区分 submit / reset / button 三种按钮作用；
  7. 慎用 disabled，禁用后控件数据不会提交。

总结

  1. 表单容器：<form action="" method=""> 负责整体提交；
  2. 核心控件：text /password/radio /checkbox/select /textarea/button；
  3. 关键属性：name /value/placeholder /checked/selected /disabled；
  4. 优化语义与体验：label 点击绑定、fieldset+legend 分组；
  5. 开发规范：结构清晰、语义化书写、控件命名规范，为后端数据交互打基础。
