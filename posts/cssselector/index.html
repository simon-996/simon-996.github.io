html-css-6-选择器详解

发表于2026-04-27|更新于2026-04-27|课件
|浏览量:

🎯 学习目标

  1. 掌握复合选择器的使用（交集、并集）。
  2. 理解伪类选择器的作用，能区分动态伪类与结构伪类。
  3. 熟练使用关系选择器精准定位元素（后代、子代、相邻兄弟、通用兄弟）。
  4. 了解伪元素选择器的语法与应用场景。
  5. 掌握属性选择器的常见用法。
  6. 能综合运用各类选择器完成页面样式控制。

复合选择器

基础选择器（标签、类、ID、通配符）一次只能选一种元素。复合选择器将多个基础选择器组合，实现更精准的选中。

交集选择器

同时满足多个条件的元素。选择器之间紧挨着写，中间不加空格。

注意：交集选择器中，标签名必须写在最前面。

并集选择器（分组选择器）

同时选中多组元素，共用同一套样式。选择器之间用逗号 , 分隔。

适用场景：多个元素样式相同时，避免重复书写。

关系选择器

根据元素之间的层级关系来选中目标元素。

先看一段 HTML 结构，后续选择器都基于它讲解：

后代选择器

选中某元素所有后代中匹配的元素，不管嵌套多深。用空格连接。

子代选择器

只选中某元素的直接子元素，不选更深层级。用 > 连接。

对比：.container p 会选中所有后代的 <p>，.container > p 只选直接子级的 <p>。

相邻兄弟选择器

选中紧挨在某元素后面的第一个兄弟元素。用 + 连接。

通用兄弟选择器

选中某元素后面的所有指定兄弟元素。用 ~ 连接。

四种关系选择器对比

选择器 符号 选中范围 记忆口诀
后代 空格 所有后代，不管多深 全家后代
子代 > 仅直接子元素 亲生子女
相邻兄弟 + 紧挨着的下一个兄弟 隔壁邻居
通用兄弟 ~ 后面所有的指定兄弟 所有弟弟

伪类选择器

伪类用于选中元素的某种状态，不是页面中真实存在的类，而是浏览器根据条件动态添加的。

语法：选择器后面加 : 冒号。

动态伪类（链接与交互）

最常用于 <a> 标签和交互元素：

伪类 说明 示例
:link 未访问过的链接 a:link { color: blue; }
:visited 已访问过的链接 a:visited { color: purple; }
:hover 鼠标悬停时 a:hover { color: red; }
:active 鼠标按下未松开时 a:active { color: orange; }
:focus 元素获得焦点时 input:focus { border-color: blue; }

重要规则：链接伪类必须按 LVHA 顺序书写，否则会失效：

:link → :visited → :hover → :active

记忆口诀：LoVe HAte（爱恨原则）

结构伪类

根据元素在父元素中的位置来选中，常用于列表、表格等场景。

伪类 说明 示例
:first-child 父元素的第1个子元素 li:first-child { font-weight: bold; }
:last-child 父元素的最后1个子元素 li:last-child { border: none; }
:nth-child(n) 父元素的第n个子元素 li:nth-child(2) { color: red; }
:nth-child(odd) 所有奇数位子元素 tr:nth-child(odd) { background: #eee; }
:nth-child(even) 所有偶数位子元素 tr:nth-child(even) { background: #fff; }

nth-child 的 n 值写法

nth-child() 的括号里非常灵活：

写法 含义
2 第2个
odd / 2n+1 奇数位（第1、3、5…个）
even / 2n 偶数位（第2、4、6…个）
3n 第3、6、9…个
3n+1 第1、4、7…个
-n+3 前3个（第1、2、3个）

伪元素选择器

伪元素用于选中元素中的某个虚拟部分（如第一行文字、前面插入内容），不是真实存在的 HTML 元素。

语法：选择器后面加 :: 双冒号（CSS3 规范，兼容旧写法单冒号）。

伪元素 说明 示例
::before 在元素内容前面插入虚拟内容 .box::before { content: "★"; }
::after 在元素内容后面插入虚拟内容 .link::after { content: "→"; }
::first-letter 选中第一个字符 p::first-letter { font-size: 2em; }
::first-line 选中第一行文字 p::first-line { font-weight: bold; }

注意：::before 和 ::after 必须设置 content 属性，否则不会显示。

属性选择器

根据 HTML 元素的属性名和属性值来选中元素。

选择器 说明 示例
[attr] 含有指定属性的元素 [title] { cursor: help; }
[attr="val"] 属性值完全等于 val input[type="text"] { width: 200px; }
[attr^="val"] 属性值以 val 开头 a[href^="https"] { color: green; }
[attr$="val"] 属性值以 val 结尾 a[href$=".pdf"] { color: red; }
[attr*="val"] 属性值包含 val a[href*="github"] { color: #333; }

选择器优先级（权重）补充

第4课讲过基础选择器的权重：ID(100) > 类(10) > 标签(1)。现在补充复合选择器的权重计算。

权重叠加规则

复合选择器的权重 = 各个基础选择器权重之和。

选择器 计算过程 总权重
p 1 1
.nav 10 10
#header 100 100
p.highlight 1 + 10 11
div p 1 + 1 2
#header .nav p 100 + 10 + 1 111
ul > li:first-child 1 + 1 + 10 12

特殊规则

  • **!important**：强制最高优先级，写在属性值后面。慎用，会破坏权重体系。
  • 行内样式：权重 1000，高于所有选择器。
  • **通配符 ***：权重为 0，不参与叠加。
  • 继承的样式：没有任何权重，比最低选择器权重还低。

综合练习

完成以下需求，写出对应的选择器：

需求：

  1. 选中 nav 下所有链接：.nav a
  2. 选中 main 的直接子元素 h2：main > h2
  3. 选中 intro 类的段落：p.intro
  4. 选中紧接 h2 后面的段落：h2 + p
  5. 选中以 https 开头的链接：a[href^="https"]
  6. 选中以 .pdf 结尾的链接：a[href$=".pdf"]
  7. 选中 main 下偶数位的 p：main p:nth-child(even)

开发易错点总结

  1. 后代与子代混淆：div p 选所有后代，div > p 只选直接子元素。注意区分。
  2. LVHA 顺序错误：链接伪类不按顺序写会导致样式被覆盖失效。
  3. ::before/::after 缺少 content：不写 content 属性伪元素不会显示，即使设了其他样式也没用。
  4. nth-child 计数从1开始：不是从0开始，nth-child(1) 是第一个。
  5. 伪类与伪元素符号混淆：伪类用单冒号 :hover，伪元素用双冒号 ::before。
  6. 属性选择器引号：属性值需要加引号，[type="text"] 而不是 [type=text]。

总结

  1. 复合选择器：交集（紧挨）精准筛选，并集（逗号）批量设样式。
  2. 关系选择器：空格（后代）、>（子代）、+（相邻兄弟）、~（通用兄弟）。
  3. 伪类选择器：动态伪类控制交互状态（LVHA顺序），结构伪类按位置选中。
  4. 伪元素选择器：::before/::after 插入内容，必须设置 content。
  5. 属性选择器：按属性名或属性值筛选，^开头 $结尾 *包含。
  6. 权重叠加：复合选择器权重相加，!important 慎用。
