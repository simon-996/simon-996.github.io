html-css-1-网页基本结构标签

发表于2026-03-06|更新于2026-03-11|课件
|浏览量:

🎯 学习目标

  1. 理解 HTML 的基本文档结构。
  2. 掌握双标签与单标签的区别。
  3. 学会使用常用的文本标签（标题、段落、样式）。
  4. 核心理念：理解什么是“语义化”，并养成良好习惯。

1. HTML 简介

HTML (Hyper text Markup Language) = 超文本标记语言。

  • 超文本：不仅仅是文字，还可以包含图片、链接、视频等。
  • 标记语言：使用标签（Tags）来标记内容的含义。
  • 核心符号：< > (尖括号)。

1.1 HTML 基本骨架 每个 HTML 文件都必须包含以下结构：

标签 作用 说明
<!DOCTYPE html> 声明文档类型 必须放在第一行，告诉浏览器这是 HTML5。
<html> 根元素 包裹整个网页，lang="zh-CN" 表示中文。
<head> 头部信息 存放不显示在页面上的配置（编码、标题、CSS 链接）。
<meta> 元数据 关键属性：charset="UTF-8" (防止乱码)。
<title> 页面标题 显示在浏览器标签页上的文字。
<body> 主体内容 存放显示在页面上的所有内容（文字、图片等）。

💡 小贴士：注释标签
在代码中写备注，浏览器不会显示，方便开发者阅读。

2. 标签（Tags）详解

2.1标签的组成

符号：标签用<>尖括号表示

组成：开始标签、内容、结束标签

别称：标签也称为 元素

2.2 标签的分类

大部分标签都是双标签，少部分单标签

双标签 (Paired Tags)

有开始也有结束，内容写在中间。绝大多数标签都是双标签。

单标签 (Self-Closing Tags)

只有一个标签，没有结束标签，也不能包含内容。

⚠️ 注意：在 HTML5 中，末尾的 / 可以省略（如 <br>），但写上也是合法的。

2.3 标签的关系

并列关系：两个标签互不包含，平级排列。

嵌套关系：一个标签包含另一个标签（子标签必须在父标签内部闭合）。

3. 常用文字标签

3.1 段落与标题

标签 英文全称 作用 特点
<p> Paragraph 定义段落 独占一行，上下有默认间距。
<h1>~`
`
 Heading 定义标题 h1 最大（最重要），h6 最小。一个页面建议只有一个 h1。

3.2 字体样式标签 (重点：语义化)

HTML 不仅是为了“好看”，更是为了表达“含义”。

标签 作用 语义说明 推荐度
<b> 加粗 纯样式加粗，无特殊含义 ⭐⭐
<strong> 加粗 表示“重要/强调”，搜索引擎会重视 ⭐⭐⭐⭐⭐
<i> 斜体 纯样式斜体，无特殊含义 ⭐⭐
<em> 斜体 表示“语气强调” ⭐⭐⭐⭐⭐
<u> 下划线 表示专有名词等（慎用，易与链接混淆） ⭐⭐
<s> / <del> 删除线 表示已失效、降价等内容 ⭐⭐⭐⭐
<font> 自定义字体 已过时！ 请用 CSS 代替 ❌ 禁止使用

4. 核心理念：HTML 语义化

什么是语义化？
根据内容的架构和含义选择恰当的 HTML 标签，而不是只为了“长得像”。
原则：让标签回归「含义」，让 CSS 负责「样式」。

为什么要语义化？

  1. **SEO (搜索引擎优化)**：搜索引擎（如百度、Google）通过 h1 知道文章标题，通过 p 知道正文，合理的结构能提升排名。
  2. 可读性与维护：开发者看到 <h2> 就知道是大章节，看到 <nav> 就知道是导航，代码一目了然。
  3. 无障碍访问：屏幕阅读器（视障人士使用）能准确朗读出“这是标题”、“这是强调内容”。

总结

  • 结构：<!DOCTYPE> + html + head + body。
  • 标签：分清双标签和单标签。
  • 文本：标题用 h1-h6，段落用 p，强调用 strong/em。
  • 原则：拒绝 <font>，坚持语义化写作。
