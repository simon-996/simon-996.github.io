🎯 学习目标

  1. 理解 CSS 的核心作用：结构与表现分离。
  2. 掌握 CSS 的基础语法结构。
  3. 熟练掌握 CSS 的三种引入方式及优先级。
  4. 能够区分并熟练使用标签、类、ID 及通配符选择器。

CSS 简介

什么是 CSS？

CSS 全称是Cascading Style Sheets - 层叠样式表。

  • 比喻：
    • 如果 HTML 是网页的“骨架”和“肉体”（“毛坯房”）；
    • 那么 CSS 就是网页的“皮肤”和“妆容”（“精装修”）。
  • 分工：
    • HTML：负责结构（标题、段落、图片）。
    • CSS：负责表现（颜色、字体、布局、动画）。

为什么要使用 CSS？

  • 结构与表现分离：HTML 专注于语义化，CSS 专注于美化，便于维护和改版。
  • 提高开发效率：一套 CSS 代码可以控制多个页面的样式。
  • 提升用户体验：实现更丰富的视觉效果和响应式布局。

CSS 基础语法

CSS 的核心就是告诉浏览器“谁应该变成什么样”。

语法结构

  • 选择器：指定要改变样式的 HTML 元素。
    • 例如上面的 p，表示选中页面中所有的 <p> 标签。
  • 声明块：花括号 {} 包裹的部分。
  • 属性：想要改变的样式特征。
    • 例如 color 代表文字颜色。
  • 值：属性的具体设定。
    • 例如 red 代表红色。

️ 注意：

  • 属性与值之间用英文冒号 : 分隔。
  • 每条声明之间必须用分号 ; 分隔（最后一行建议也加上，防止后续添加代码出错）。

CSS 引入方式

写好了 CSS 代码，我们需要把它“放入”HTML 页面中才能生效。主要有三种方式。

方式 语法位置 代码示例 特点
行内样式 标签的 style 属性 <h1 style="color:red;">标题</h1> 优先级最高，但结构与样式未分离，不推荐使用。
内部样式 HTML <head> 中的 <style> 标签 <style>p { color: red; }</style> 仅对当前页面有效，适合测试或单页专用样式。
外部样式 引入独立的 .css 文件 <link rel="stylesheet" href="style.css"> 推荐。结构与样式完全分离，可复用，易于维护。

避坑指南：

  • 使用外部样式表时，<link> 标签必须包含 rel="stylesheet" 属性，否则浏览器无法识别这是样式表，导致样式加载失败。

小贴士：实际开发中，99% 的情况都使用外部样式表。

CSS 选择器

学会了语法和引入方式，我们需要更精准地控制页面元素样式。选择器就是用来“选中”元素的工具。

CSS 选择器有：

  1. 基础选择器
  2. 关系选择器
  3. 分组选择器
  4. 伪类和伪元素
  5. 属性选择器

基础选择器

由单个选择器组成，常见的有：

选择器类型 语法符号 示例代码 说明
标签选择器 标签名 p { color: blue; } 选中页面中所有的 <p> 标签。
类选择器 .类名 .nav { width: 100px; } 选中 class="nav" 的元素，最常用。
ID 选择器 #ID名 #header { height: 60px; } 选中 id="header" 的元素，具有唯一性。
通配符选择器 * * { margin: 0; padding: 0; } 选中页面中所有元素，常用于初始化。

选择器优先级（权重）

当一个元素被多个选择器（或多种引入方式）选中时，浏览器会根据优先级决定使用哪个样式。

权重计算规则（分值法）

我们可以给不同的选择器赋予不同的“分值”，总分越高，优先级越高：

  • ID 选择器：100 分
  • 类选择器：10 分
  • 标签选择器：1 分

举例计算：

  • #header .nav = 100 + 10 = 110 分 (胜出)
  • div .nav = 1 + 10 = 11 分

简单记忆口诀：行内 > ID > 类 > 标签

引入方式与优先级的关系

  • 就近原则：在引入方式中，行内样式优先级最高。
  • 后来者居上：如果是同一种引入方式（如都是内部样式），且选择器权重相同，后写的样式会覆盖先写的样式。

颜色

在 CSS 中，颜色可以通过多种方式定义，其中最常用的是颜色关键字、十六进制值和 RGB 函数。它们最终在浏览器中都会被解析为相同的颜色值，但在语法、可读性和功能上各有不同。

颜色关键字

颜色关键字是直接用颜色的英文名称来定义颜色，例如 red、blue、orange。

  • 优点：可读性极高，代码简洁。

  • 缺点：能表示的颜色非常有限。CSS 规范定义了超过 160 种具名颜色，但远不能满足所有设计需求。

十六进制 (Hexadecimal)

十六进制颜色值以 # 开头，后跟 3 位或 6 位十六进制数（0-9, a-f），分别代表红、绿、蓝三个颜色分量。

  • **#RRGGBB**：标准写法，两位十六进制数表示一个颜色分量，范围从 00 (0) 到 FF (255)。
  • **#RGB**：简写形式。当每个颜色分量的两位数字都相同时（如 #ff0000），可以简写为 #f00。
  • 优点：表达的颜色范围非常广泛，是 Web 开发中最常用的颜色表示法之一。
  • 缺点：不如 RGB 函数直观，无法直接看出各个颜色分量的具体数值。

RGB 函数

RGB 函数通过 rgb() 或 rgba() 来定义颜色，直接指定红(R)、绿(G)、蓝(B)三个分量的强度，还可以指定透明度（Alpha 通道）。

  • 语法 rgb(R, G, B) 或 rgba(R, G, B, A)
    • R, G, B：可以是 0 到 255 之间的整数，也可以是 0% 到 100% 的百分比。
    • **A (Alpha)**：表示透明度，范围从 0（完全透明）到 1（完全不透明）。
  • 优点：
    • 直观：可以清晰地看到每个颜色通道的值。
    • 功能强大：原生支持透明度 (rgba) 和百分比表示。
    • 可编程性：在现代 CSS 中，rgb() 函数支持更高级的语法，如 rgb(from ...)，可以基于其他颜色进行计算。

答案：

  1. <link> 缺少 rel="stylesheet"。
  2. color 后面用了中文冒号 ：，且 red 后面漏了分号 ;。
  3. HTML 中是 id="header"，CSS 中应该用 #header，而不是 .header。

开发易错点总结

  1. 符号错误：CSS 中使用英文冒号 : 和分号 ;，漏写分号会导致后续样式失效。
  2. 选择器拼写：类名和 ID 名区分大小写，拼写必须与 HTML 中完全一致。
  3. 路径问题：引入外部 CSS 文件时，href 路径写错会导致样式加载失败（404）。
  4. 滥用 ID：ID 权重太高，不利于后期维护，建议尽量使用类选择器。
  5. 单位遗漏：除 0 以外，大多数数值属性（如 width, font-size）必须带单位（如 px）。

总结

  1. CSS 核心：负责网页美化，实现结构与表现分离。
  2. 语法基础：选择器 { 属性: 值; }，注意英文标点。
  3. 引入方式：首选外部样式表 (<link>)，注意 rel 属性。
  4. 选择器：熟练掌握 .类名 和 #ID名 的区别与使用场景。
  5. 优先级：理解 行内 > ID > 类 > 标签 的层叠规则。
